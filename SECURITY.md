# Security Policy

## Supported Versions

This project is maintained as a small public collection of installable ChatGPT skills.

| Version | Supported |
|---|---|
| 1.x | Yes |
| < 1.0.0 | No |

## Reporting a Vulnerability

Please do not open a public issue for a suspected security problem.

Instead, report it privately by contacting the maintainers through the repository owner's preferred private channel. If a dedicated security contact is later added to the repository profile or organization profile, use that channel.

When reporting a vulnerability, please include:
- a short description of the issue
- affected files or skill packages
- reproduction steps
- expected impact
- any suggested remediation

## Response Goals

Maintainers should aim to:
- acknowledge receipt within 7 days
- assess severity and reproduce the issue
- prepare a fix or mitigation
- publish a coordinated disclosure when appropriate

## Scope

This policy mainly applies to:
- malicious or unsafe instructions unintentionally included in a skill
- packaging mistakes that expose unwanted files
- documentation or release processes that encourage insecure use

General quality issues, unclear prompts, or feature requests should be opened as normal GitHub issues.
